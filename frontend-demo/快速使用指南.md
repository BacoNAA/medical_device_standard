# 快速使用指南

## 🚀 三步启动

### Windows用户

1. 双击运行 `启动开发服务器.bat`
2. 等待自动安装依赖并启动（首次启动需要几分钟）
3. 浏览器将自动打开系统登录页面

### 命令行启动

```bash
# 1. 进入项目目录
cd frontend-demo

# 2. 安装依赖（首次运行）
npm install

# 3. 启动开发服务器
npm run dev
```

## 👤 登录体验

### 方式1：快速登录（推荐）

在登录页面点击快速登录按钮，一键登录不同角色：

- **数据输入员** - 体验标准导入、内容维护功能
- **系统管理员** - 体验用户管理、系统配置功能
- **业务人员** - 体验产品技术要求编辑功能
- **质量管理人员** - 体验表1导出功能
- **设备管理人员** - 体验设备管理、表2导出功能
- **实验室人员** - 体验表3导出功能

### 方式2：手动登录

输入用户名和任意密码即可登录：

| 用户名 | 角色 |
|--------|------|
| `dataadmin` | 数据输入员 |
| `sysadmin` | 系统管理员 |
| `yw001` | 业务人员 |
| `zl001` | 质量管理人员 |
| `sb001` | 设备管理人员 |
| `sy001` | 实验室人员 |

## 🎯 核心功能体验路径

### 路径1：标准数据管理（推荐首次体验）

```
登录为任意角色 → 标准浏览 → 查看GB 15810-2019详情 → 查看检验项目 → 查看附录
```

**亮点**：
- 三级数据结构展示
- 检验项目详细信息
- 方法类型标签（无独立方法/引用附录/完整方法）
- 使用设备关联展示

### 路径2：标准导入流程（数据输入员）

```
登录为dataadmin → 标准导入 → 上传文件 → 系统自动解析 → 预览确认 → 完成导入
```

**亮点**：
- 4步骤向导式流程
- 自动识别演示（绿色/黄色/红色状态标注）
- 人工审核机制

### 路径3：产品技术要求编辑（业务人员）

```
登录为yw001 → 产品技术要求编辑 → 新建文档 → 三栏编辑器 → 智能辅助 → 导出Word
```

**亮点**：
- 专业的三栏布局
- 模板插入功能
- 关键词联想查找（输入"密封性"试试）
- 智能辅助面板

### 路径4：数据导出（质量/设备/实验室人员）

```
登录对应角色 → 数据导出 → 选择标准 → 预览数据 → 导出Excel
```

**体验不同的表格**：
- 质量管理员（zl001）→ 表1：能力建设信息网站项目表
- 设备管理员（sb001）→ 表2：认可检验检测能力表
- 实验室人员（sy001）→ 表3：操作规程表

### 路径5：自定义导出（所有角色）

```
任意角色 → 自定义导出 → 创建模板 → 选择字段 → 导出数据
```

**亮点**：
- 灵活的模板配置
- 字段自由选择
- 公开/私有模板

### 路径6：设备管理（数据输入员/设备管理员）

```
登录为dataadmin或sb001 → 设备管理 → 新增设备 → 建议编号 → 保存
```

**亮点**：
- 设备编号格式验证（A001格式）
- 自动建议下一个可用编号
- 启用/停用状态管理

### 路径7：错误报告（所有用户）

```
任意角色 → 错误报告 → 提交错误报告 → 查看报告列表 → 状态筛选
```

**数据输入员可体验处理流程**：
```
登录为dataadmin → 错误报告 → 处理待处理报告 → 修正或驳回
```

### 路径8：系统管理（系统管理员）

```
登录为sysadmin → 用户管理 → 创建用户 → 重置密码 → 查看操作日志 → 系统配置
```

**亮点**：
- 完整的用户生命周期管理
- 详细的操作日志
- 丰富的系统参数配置

## 💡 使用技巧

1. **切换角色体验**：退出登录后可切换不同角色，查看不同的菜单和权限
2. **关注细节**：注意不同状态的颜色标签、提示信息
3. **数据联动**：在标准详情页查看设备，然后到设备管理页查看设备列表
4. **流程完整性**：尝试完整走完一个功能流程（如标准导入的4个步骤）

## ⚠️ 注意事项

1. **数据不持久**：刷新页面后，所有操作会重置（纯前端Demo）
2. **功能演示**：部分功能（如Word导出）仅做提示，不生成真实文件
3. **浏览器要求**：建议使用Chrome 90+、Edge 90+或Firefox 88+
