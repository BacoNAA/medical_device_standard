## 🎯 核心业务逻辑

### 1. 方法类型的三种形式

```javascript
// methodType 字段区分三种情况
1: 无独立方法
   - 项目要求中已包含检验方法
   - methodContent 为空

2: 引用附录
   - 方法单独在附录中
   - methodRef 记录引用位置（如 "A.1"）
   - 可通过 methodRef 关联到 appendices

3: 完整方法
   - methodContent 存储完整方法内容
```

### 2. 设备关联设计

```javascript
// 设备表：同类型设备只一条
{
  code: 'A001',
  name: '电子天平',
  // 不记录具体规格
}

// 项目-设备关联：记录具体规格
{
  itemId: 1,
  equipmentCode: 'A001',
  specification: '分度值0.1mg'  // 具体规格在这里
}

// 优势：避免设备信息重复
```

### 3. 导入处理状态

```javascript
// 检验项目的处理状态
status: 'success'  // 绿色：自动处理成功
status: 'warning'  // 黄色：新设备需确认
status: 'error'    // 红色：需人工补充
```

## ⚙️ 开发注意事项

### Mock数据的使用
- 所有数据都在 `mock/standardData.js` 中定义
- 数据是静态的，刷新页面会重置
- 修改Mock数据可以立即看到效果

### 路由参数
- 标准详情页使用动态路由 `/standard-detail/:id`
- 目前ID不影响显示（都显示同一个Mock数据）
- 真实环境中需要根据ID加载不同标准

### 组件自动导入
- Element Plus组件无需手动导入
- Vue API（ref, computed等）无需手动导入
- 由 `unplugin` 插件自动处理
